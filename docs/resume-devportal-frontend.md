# DevPortal 项目简历素材（前端社招）

## 项目定位（一句话）
主导 DevPortal 前端与工程化闭环建设，打通 Jenkins 构建、Webhook 触发、Apifox 接口同步与钉钉机器人通知，提升联调效率与交付稳定性。

## 一页简历精简版（可直接粘贴）
1. 主导 DevPortal 前端架构与核心模块开发，统一 Next.js + TypeScript 工程规范。
2. 打通 Jenkins 构建与 Webhook 流程，构建后自动同步 OpenAPI 到 Apifox。
3. 建立 Apifox 同步结果钉钉机器人通知机制，提升接口变更可见性与协作效率。
4. 推动前端基于 Apifox + AI 自助调试接口，降低联调成本并缩短交付周期。
5. 沉淀部署与排障文档，提升跨环境交付稳定性与团队复用效率。

## 面试展开版（技术细节 + 结果表达）
1. 作为项目主导，负责 DevPortal 前端整体方案设计与落地，基于 Next.js App Router + TypeScript 构建页面、路由与 API 调用规范，支撑多模块并行迭代。
2. 设计并落地 Jenkins -> Webhook -> OpenAPI 导出/同步 -> Apifox 自动化链路，避免接口文档滞后，保障前后端以同一契约联调。
3. 将 Apifox 同步结果接入钉钉群机器人，自动推送成功/失败状态与关键信息，减少人工同步与口头确认，问题发现时间缩短至分钟级。
4. 推动前端团队使用 Apifox + AI 自动调试接口（自动造参、快速回归），将重复手工联调转为自助化流程，显著降低沟通成本。
5. 覆盖工程交付闭环：构建、通知、接口同步、文档沉淀、部署排障，提升项目可维护性和团队协作效率。

## 30 秒项目讲述（面试可直接说）
我在这个项目里不仅负责前端页面开发，还主导了工程化协作链路。核心是把 Jenkins 构建、Webhook、OpenAPI 到 Apifox 的同步流程打通，并把同步结果实时通知到钉钉机器人。这样前端和测试始终基于最新接口契约开展联调，再结合 Apifox + AI 做自动调试，减少了大量重复沟通和手工造参，整体交付效率和稳定性都明显提升。
